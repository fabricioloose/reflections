O que acontece ao iniciar um repositório? Por que você precisa fazer isso?
R: É criado a estrutura de repositório para o Git (.git). E necessário
sua execução para que o repositório seja criado.

Em que se difere a área de preparação do diretório de trabalho e do repositório?
Qual valor você acha que ela oferece?
R: O repositorio os files e commits ja estão persistidos. Area de preparação
é um pré fase para um commit. Seu valor é importante pelo fato de proporcionar
uma analise logica para os commits.

Como você pode usar a área de preparação para garantir que tenha um commit por alteração lógica?
R: Comparando a arede de preparacao com a area de trabalho.

Quais são algumas das situações em que branches seriam úteis para manter seu histórico organizado? 
Como branches ajudariam?
R: Tradução para uma nova lingua, nova GUI, etc. Atraves de branch's pode-se reverter dos testes
para a versao de producao e tb unir branchs validos ao de producao.

Como os diagramas ajudam a visualizar a estrutura de branches?
R: Através de ramificações podemos saber onde cada commit esta, quem 
é a sua ramificacao, ou se eh um commit que ainda nao possui nenhum
branch

Qual é o resultado da mesclagem de dois branches? Por que o representamos no diagrama dessa forma?
R: Uniao com o branch master. Para facilitar o entendimento das ramificacoes dos branchs
